# 任务：录像诊断功能 UI 设计

## 基本信息
- **任务 ID**: ui-001-video-diagnosis
- **功能名称**: 录像诊断交互界面
- **优先级**: P0
- **推荐模型**: kimi-k2.5
- **参考样例**: 用户提供的截图（录像丢失详情界面）

---

## 功能需求

### 核心功能
显示录像诊断结果，包括：
1. **录像丢失详情** - 显示丢失的录像时间段、原因
2. **存储天数** - 显示录像存储时长
3. **设备名称** - 显示监控设备名称
4. **诊断状态** - 正常/异常/丢失

### 界面元素
- [ ] 设备名称显示区域
- [ ] 存储天数统计卡片
- [ ] 录像丢失详情列表
- [ ] 时间轴展示（丢失时间段）
- [ ] 状态标签（正常/异常）
- [ ] 刷新/重新诊断按钮

---

## 视觉要求

### 整体风格
- **风格**: 数据监控面板风格，清晰直观
- **配色**: 
  - 主色：#1890FF（科技蓝）
  - 正常状态：#52C41A（绿色）
  - 异常状态：#F5222D（红色）
  - 警告状态：#FAAD14（橙色）
- **背景**: 浅灰色系 (#F5F7FA)

### 布局要求
- **页面结构**: 上下结构
  - 顶部：设备信息 + 诊断状态
  - 中间：存储统计卡片
  - 底部：录像丢失详情列表
- **响应式**: 适配手机和桌面

### 交互要求
- [ ] 状态标签有颜色区分
- [ ] 列表项可点击展开详情
- [ ] 按钮有悬停效果
- [ ] 加载状态动画
