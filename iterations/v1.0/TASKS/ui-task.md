# UI 设计任务 - 录像诊断功能

## 功能描述
设计一个录像诊断功能的用户界面，用户可以上传录像文件并查看诊断结果。

## 页面需求

### 1. 上传页面
- 文件拖拽上传区域
- 支持格式提示（MP4, MOV, AVI 等）
- 上传进度条
- 开始诊断按钮

### 2. 诊断结果页面
- 录像基本信息（文件名、时长、分辨率）
- 诊断评分（0-100 分）
- 问题列表（如有）：
  - 画面模糊
  - 音频不同步
  - 帧率过低
  - 编码问题
- 修复建议

### 3. 历史记录页面
- 历史诊断记录列表
- 点击查看详情

## 输出要求
- **目录**: `designs/video-diagnosis/`
- **文件**:
  - `upload.html` - 上传页面
  - `result.html` - 诊断结果页面
  - `history.html` - 历史记录页面
  - `style.css` - 统一样式
  - `app.js` - 交互逻辑

## 依赖
- 无

## 推荐模型
- **模型**: `kimi-k2.5`
- **理由**: UI 设计任务，需要生成 HTML/CSS/JS 代码

## 技术建议
- 响应式设计（支持桌面和移动）
- 使用现代 CSS（Flexbox/Grid）
- 文件上传使用 AJAX

## 验收标准
- 页面美观、易用
- 上传区域有明显的视觉反馈
- 诊断结果清晰展示问题和评分
- 代码可直接在浏览器中运行
