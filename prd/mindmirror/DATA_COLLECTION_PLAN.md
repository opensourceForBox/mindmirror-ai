## 1. 概述

### 1.1 目标

为 MindMirror AI 情绪识别模型训练提供高质量、多样化的标注数据，支持 9 类情绪分类（含焦虑/抑郁心理状态）及 valence/arousal 维度评估。

### 1.2 数据需求

| 数据类型 | 目标数量 | 用途 | 优先级 |
|---------|---------|------|--------|
| 面部图像 | 50,000 张 | 视频情绪识别模型训练 | P0 |
| 音频片段 | 20,000 段 | 音频情绪识别模型训练 | P0 |
| 音视频同步 | 10,000 段 | 多模态融合模型训练 | P0 |
| 文本对话 | 30,000 条 | 文本情绪分析备用 | P1 |

## 2. 数据标注规范

### 2.1 情绪标签定义

#### 2.1.1 基本情绪（Ekman 6 类）

**快乐 (happy)**
- 面部特征：颧大肌收缩（嘴角上扬）、眼轮匝肌收缩（鱼尾纹）
- 音频特征：基频较高、能量中等、语速适中
- 触发场景：积极事件回忆、成功体验、愉快对话

**悲伤 (sad)**
- 面部特征：口角降肌收缩（嘴角下垂）、眉毛内角上扬
- 音频特征：基频低、能量低、语速缓慢、停顿多
- 触发场景：失落事件回忆、分离场景、挫折经历

**愤怒 (angry)**
- 面部特征：眉间肌收缩（眉毛下压）、嘴唇紧绷、下颌前伸
- 音频特征：基频变化大、能量高、语速快、音量大
- 触发场景：不公平事件、被冒犯、目标受阻

**恐惧 (fearful)**
- 面部特征：上眼睑提升、眉毛上扬并聚拢、嘴唇横向拉伸
- 音频特征：基频高、能量变化大、语速快、呼吸声明显
- 触发场景：威胁场景、未知风险、紧张情境

**惊讶 (surprised)**
- 面部特征：眉毛高抬、眼睛睁大、下颌下落
- 音频特征：基频突变、能量突变、语调上扬
- 触发场景：意外事件、突发消息、出乎意料

**厌恶 (disgusted)**
- 面部特征：鼻子皱起、上唇提升、下唇可能前伸
- 音频特征：基频低、能量低、语调低沉
- 触发场景：令人不适的事物、道德违背、生理厌恶

#### 2.1.2 心理状态扩展（2 类）

**焦虑 (anxious)**
- 面部特征：眉毛微皱、眼部紧张、嘴唇紧闭或咬唇
- 音频特征：基频偏高、语速快、停顿频繁、填充词多（"嗯"、"啊"）
- 触发场景：压力情境、不确定性、担忧未来
- 持续时间：通常持续较长时间（>30 秒）

**抑郁 (depressed)**
- 面部特征：表情减少、眼神呆滞、面部肌肉松弛
- 音频特征：基频低、能量低、语速缓慢、音量低、单调
- 触发场景：无助感、无望感、自我否定
- 持续时间：长期状态（分钟级）

#### 2.1.3 中性状态

**中性 (neutral)**
- 面部特征：面部肌肉放松、无明显表情
- 音频特征：基频中等、能量中等、语调平稳
- 触发场景：平静状态、日常对话、无明显情绪刺激

### 2.2 Valence/Arousal 标注规范

#### 2.2.1 Valence（效价）

衡量情绪的正负向程度，范围 [-1.0, 1.0]

| 分值范围 | 描述 | 示例情绪 |
|---------|------|----------|
| 0.8 ~ 1.0 | 非常积极 | 狂喜、兴奋 |
| 0.5 ~ 0.8 | 积极 | 快乐、满足 |
| 0.2 ~ 0.5 | 略积极 | 平静、安心 |
| -0.2 ~ 0.2 | 中性 | 中性状态 |
| -0.5 ~ -0.2 | 略消极 | 担忧、不安 |
| -0.8 ~ -0.5 | 消极 | 悲伤、焦虑 |
| -1.0 ~ -0.8 | 非常消极 | 绝望、极度愤怒 |

#### 2.2.2 Arousal（唤醒度）

衡量情绪的生理激活程度，范围 [0.1, 1.0]

| 分值范围 | 描述 | 示例情绪 |
|---------|------|----------|
| 0.8 ~ 1.0 | 非常高唤醒 | 狂喜、极度愤怒、恐惧 |
| 0.6 ~ 0.8 | 高唤醒 | 惊讶、焦虑、兴奋 |
| 0.4 ~ 0.6 | 中等唤醒 | 一般快乐、轻微愤怒 |
| 0.2 ~ 0.4 | 低唤醒 | 平静、轻微悲伤 |
| 0.1 ~ 0.2 | 非常低唤醒 | 抑郁、疲惫、麻木 |

### 2.3 标注质量标准

#### 2.3.1 标注准确性要求

| 指标 | 要求 | 测量方式 |
|------|------|----------|
| 情绪分类准确率 | ≥85% | 与金标准对比 |
| Valence 标注误差 | ≤0.15 | 绝对误差均值 |
| Arousal 标注误差 | ≤0.15 | 绝对误差均值 |
| 边界案例处理 | ≥80% | 专家仲裁一致率 |

#### 2.3.2 标注完整性要求

- 每个样本必须标注：主情绪标签、valence、arousal
- 可选标注：次级情绪（如有）、置信度、备注
- 问题样本标记：模糊、多情绪、质量差

#### 2.3.3 标注时效要求

| 数据类型 | 单样本标注时间 | 批次交付周期 |
|---------|--------------|-------------|
| 静态图像 | 15-30 秒 | 3-5 个工作日 |
| 音频片段 (5 秒) | 30-60 秒 | 5-7 个工作日 |
| 音视频同步 (10 秒) | 60-90 秒 | 7-10 个工作日 |

### 2.4 标注一致性检验方法

#### 2.4.1 Cohen's Kappa 系数

用于衡量两名标注员之间的一致性：

```
κ = (p₀ - pₑ) / (1 - pₑ)

其中:
- p₀: 实际一致率
- pₑ: 期望一致率（随机情况下）
```

**标准**:
- κ ≥ 0.8: 优秀一致性
- 0.6 ≤ κ < 0.8: 良好一致性
- 0.4 ≤ κ < 0.6: 中等一致性
- κ < 0.4: 一致性不足，需重新培训

#### 2.4.2 Krippendorff's Alpha

用于多名标注员的一致性检验（支持多类别）：

```
α = 1 - (Dₒ / Dₑ)

其中:
- Dₒ: 观测到的差异
- Dₑ: 期望差异
```

**标准**:
- α ≥ 0.8: 可接受用于高风险应用
- α ≥ 0.667: 可接受用于研究
- α < 0.667: 需改进标注规范或培训

#### 2.4.3 一致性检验流程

```
1. 随机抽取 10% 样本进行多人标注
2. 计算 Kappa/Alpha 系数
3. 若系数 < 0.7:
   - 分析分歧样本
   - 更新标注规范
   - 重新培训标注员
4. 若系数 ≥ 0.7:
   - 继续标注
   - 定期抽检（每 1000 样本）
```

## 4. 数据质量控制流程

### 4.1 标注员培训方案

#### 4.1.1 培训阶段

**阶段 1: 理论学习（2 天）**
- 情绪心理学基础（Ekman 理论、valence/arousal 模型）
- 9 类情绪的定义和识别要点
- 标注工具使用培训
- 隐私保护和数据安全意识

**阶段 2: 实践练习（3 天）**
- 金标准样本练习（100 个样本）
- 每日测试：准确率需≥80%
- 错题分析和反馈

**阶段 3: 试标注（2 天）**
- 真实样本标注（500 个样本）
- 双人盲标，计算一致性
- Kappa ≥ 0.7 方可上岗

**阶段 4: 持续培训（每周）**
- 每周案例分析会（1 小时）
- 新场景/边界案例讨论
- 标注规范更新培训

#### 4.1.2 培训材料

| 材料类型 | 内容 | 数量 |
|---------|------|------|

#### 4.1.3 培训考核标准

| 考核项 | 合格标准 | 优秀标准 |

### 4.2 多人标注 + 仲裁机制

#### 4.2.1 标注流程

```
样本 → 标注员 A → 标注员 B → 一致性检查 → 仲裁（如需要）→ 金标准
       ↓          ↓          ↓
    独立标注   独立标注   Kappa 计算
```

#### 4.2.2 人员配置

| 角色 | 人数 | 职责 | 资质要求 |
|------|------|------|---------|
| 标注员 | 5-8 人 | 初轮标注 | 培训考核通过 |
| 质检员 | 2 人 | 一致性检查、抽检 | 标注员 + 额外培训 |
| 仲裁员 | 1-2 人 | 分歧仲裁 | 心理学背景 + 经验 |
| 项目经理 | 1 人 | 进度管理、质量监控 | 项目管理经验 |

#### 4.2.3 仲裁机制

**触发条件**:
- 两名标注员情绪标签不一致
- Valence/arousal 差异 > 0.3
- 标注员标记为"模糊"或"不确定"

**仲裁流程**:
1. 仲裁员独立审核分歧样本
2. 参考上下文信息（如对话内容）
3. 做出最终判定
4. 记录仲裁原因（用于后续培训）

**仲裁质量监控**:
- 随机抽取 10% 仲裁样本由专家复核
- 仲裁一致率需≥90%

### 4.3 数据清洗规则

#### 4.3.1 自动清洗规则

**图像质量检查**:
```python
def check_image_quality(image):
    # 分辨率检查
    if image.width < 640 or image.height < 480:
        return False, "分辨率过低"
    
    # 模糊检测（Laplacian 方差）
    if laplacian_variance(image) < 100:
        return False, "图像模糊"
    
    # 亮度检查
    if mean_brightness(image) < 30 or mean_brightness(image) > 220:
        return False, "亮度过暗/过曝"
    
    # 人脸检测
    if not detect_face(image):
        return False, "未检测到人脸"
    
    # 人脸角度检查
    if face_yaw_angle(image) > 45 or face_pitch_angle(image) > 30:
        return False, "人脸角度过大"
    
    return True, "合格"
```

**音频质量检查**:
```python
def check_audio_quality(audio):
    # 信噪比检查
    if snr(audio) < 15:
        return False, "信噪比过低"
    
    # 静音检测
    if silence_ratio(audio) > 0.5:
        return False, "静音比例过高"
    
    # 削波检测
    if clipping_ratio(audio) > 0.05:
        return False, "音频削波"
    
    # 时长检查
    if duration(audio) < 2 or duration(audio) > 30:
        return False, "时长不合适"
    
    return True, "合格"
```

#### 4.3.2 人工清洗规则

**标注一致性检查**:
- 多人标注 Kappa < 0.5 的样本 → 重新标注或剔除
- 仲裁后仍存疑的样本 → 剔除

**异常值检测**:
- Valence/arousal 与情绪标签明显不符 → 复核或剔除
- 标注时间过短（<5 秒/样本）→ 复核

#### 4.3.3 数据清洗流程

```
原始数据 → 自动清洗 → 人工清洗 → 质量抽检 → 入库
            ↓           ↓          ↓
        剔除不合格   复核可疑    随机抽检 10%
```

#### 4.3.4 清洗记录

建立数据清洗日志，记录：
- 剔除样本 ID 及原因
- 复核样本及结果
- 清洗统计（每日/每周报告）

## 5. 隐私保护方案

### 5.1 人脸脱敏方法

#### 5.1.1 脱敏策略

**研究用数据**（模型训练）:
- 保留面部特征（情绪识别需要）
- 去除身份标识信息
- 技术：面部特征扰动 + 元数据脱敏

**公开数据**（数据集发布）:
- 完全匿名化
- 技术：面部模糊/替换 + 元数据删除

#### 5.1.2 技术实现

**方案 A: 面部特征扰动（推荐用于训练数据）**

```python
def face_anonymization(image, intensity=0.3):
    """
    轻微扰动面部特征，保留情绪信息但降低身份识别性
    """
    # 1. 检测面部关键点（68 点）
    landmarks = detect_landmarks(image)
    
    # 2. 对关键点添加微小扰动
    perturbed_landmarks = landmarks + np.random.normal(0, intensity, landmarks.shape)
    
    # 3. 基于扰动后的关键点重建面部
    anonymized_image = warp_face(image, landmarks, perturbed_landmarks)
    
    return anonymized_image
```

**方案 B: 面部模糊（用于公开数据）**

```python
def face_blur(image, blur_strength=15):
    """
    对面部区域进行高斯模糊
    """
    # 1. 检测面部区域
    face_rect = detect_face_bbox(image)
    
    # 2. 对面部区域应用高斯模糊
    image[face_rect] = cv2.GaussianBlur(image[face_rect], (blur_strength, blur_strength), 0)
    
    return image
```

**方案 C: 面部替换（用于公开数据）**

```python
def face_replacement(image, template_db):
    """
    用相似表情的模板面部替换原面部
    """
    # 1. 提取原面部特征和表情
    emotion = recognize_emotion(image)
    
    # 2. 从模板库选择相似表情的匿名面部
```

### 5.2 语音变声处理

#### 5.2.1 变声策略

**目标**: 保留情绪信息（语调、节奏、能量），去除身份特征（音色）

#### 5.2.2 技术实现

**方案 A: 音高平移（推荐）**

```python
def voice_anonymization(audio, semitones=3):
    """
    平移音高，保留语速和能量特征
    """
    # 1. 基频估计
    f0 = estimate_f0(audio)
    
    # 2. 平移基频（±3 个半音）
    f0_shifted = f0 * (2 ** (semitones / 12))
    
    # 3. 保持包络不变，重建音频
    anonymized_audio = reconstruct_audio(audio, f0_shifted)
    
    return anonymized_audio
```

**方案 B: 音色转换**

```python
def voice_conversion(audio, target_speaker):
    """
    使用 VC 技术转换到目标说话人音色
    """
    # 1. 提取内容特征（保留）
    content_features = extract_content(audio)
    
    # 2. 提取说话人特征（替换）
    speaker_features = get_speaker_embedding(target_speaker)
    
    # 3. 合成新音频
    converted_audio = synthesize_audio(content_features, speaker_features)
    
    return converted_audio
```

**方案 C: 音频扰动**

```python
def audio_perturbation(audio):
    """
    添加微小扰动，降低声纹识别准确率
    """
    # 1. 添加轻微背景噪声
    noisy_audio = audio + gaussian_noise(sn=40)
```

### 5.3 数据匿名化流程

#### 5.3.1 匿名化级别

| 级别 | 应用场景 | 处理方式 |
|------|---------|---------|
| L1: 内部训练 | 模型训练 | 面部轻微扰动 + 语音变声 |
| L2: 合作研究 | 学术合作 | 面部中度扰动 + 语音变声 + 元数据删除 |
| L3: 公开数据集 | 公开发布 | 面部模糊/替换 + 语音变声 + 元数据删除 + k-匿名 |

#### 5.3.2 K-匿名化处理

确保数据集中每个样本至少有 k-1 个其他样本在准标识符上不可区分：

```python
def k_anonymity(data, k=5, quasi_identifiers=['age_group', 'gender', 'region']):
    """
    实现 k-匿名化
    """
    # 1. 泛化准标识符
    data['age_group'] = generalize_age(data['age'], bins=['18-25', '26-35', '36-50', '50+'])
    data['region'] = generalize_region(data['city'], level='province')
    
    # 2. 检查 k-匿名性
    while not check_k_anonymity(data, k, quasi_identifiers):
        # 3. 进一步泛化或抑制
        data = generalize_or_suppress(data, quasi_identifiers)
    
    return data
```

#### 5.3.3 匿名化流程

```
原始数据 → 身份标识删除 → 面部/语音脱敏 → k-匿名化 → 质量检查 → 匿名数据集
    ↓           ↓              ↓            ↓          ↓
  用户 ID    姓名/手机号    面部扰动     准标识符   重识别风险
  联系方式   地址等         语音变声     泛化       评估<5%
```

#### 5.3.4 重识别风险评估

**评估方法**:
1. 使用脱敏后数据尝试重新识别身份
2. 重识别准确率应 < 5%
3. 定期（每季度）进行重识别攻击测试

### 8.3 情绪诱发方案

#### 8.3.1 情绪诱发方法

**视频诱发法**（推荐）:
- 播放标准化情绪诱发视频片段
- 每段 2-3 分钟，针对特定情绪
- 录制被试者的自然反应

**推荐视频库**:
| 情绪 | 视频类型 | 示例 |
|------|---------|------|
| 快乐 | 喜剧片段、可爱动物 | 《老友记》搞笑片段 |
| 悲伤 | 悲剧电影片段、感人故事 | 《忠犬八公》片段 |
| 愤怒 | 不公平场景、冲突画面 | 社会新闻片段 |
| 恐惧 | 恐怖电影片段、惊悚场景 | 恐怖片 jump scare |
| 惊讶 | 魔术表演、意外画面 | 魔术揭秘视频 |
| 厌恶 | 令人不适的画面 | 卫生条件差的场景 |
| 焦虑 | 压力情境模拟 | 公开演讲场景 |
| 抑郁 | 无助感诱发 | 孤独老人纪录片 |

**情境诱发法**:
- 设计互动任务诱发特定情绪
- 如：限时难题（焦虑）、回忆创伤（悲伤）

**回忆诱发法**:
- 引导被试者回忆特定情绪经历
- 录制回忆过程中的表情和语音
