# 产品需求文档 - Todo API 服务

## 背景与目标
创建一个简单的待办事项管理 API 服务，支持增删改查基本功能。

## 用户故事
1. 作为用户，我想要创建新的待办事项，以便记录需要完成的任务
2. 作为用户，我想要查看待办事项列表，以便了解所有任务
3. 作为用户，我想要标记待办事项为完成/未完成，以便跟踪进度
4. 作为用户，我想要删除待办事项，以便清理已完成的任务

## 验收标准
- Given 用户发送创建请求，When 请求包含标题，Then 返回创建的待办事项（含 ID）
- Given 用户请求列表，When 有已创建的待办事项，Then 返回所有待办事项
- Given 用户请求更新，When 待办事项存在，Then 更新成功并返回最新数据
- Given 用户请求删除，When 待办事项存在，Then 删除成功并返回确认

## 技术栈
- 后端：Python Flask
- 数据存储：SQLite（轻量级，无需额外部署）
- API 风格：RESTful
